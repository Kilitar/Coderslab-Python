## Next steps

Congratulations! Now you know how to create HTTP requests and extract information from them. You will be able to script the browser in such a way that it behaves completely as if you were the one controlling it.

You're probably wondering how to use this in practice. Maybe you're dreaming up plans to script your work so that it gets done on its own while you're spending all day on reddit. Or you may script your browser so that it's Python that browses reddit for you, thus saving you time - which you will spend doing your work.

No matter how invincible you feel, what we've covered in class is just the tip of the iceberg. There are entire forums, books and frameworks devoted to the topic of web scraping. These are the most interesting:

- Forums:
  
  
  
  - [webscraper.io](https://forum.webscraper.io/)
- Books:
  
  
  
  - **Web Scraping with Python: Collecting Data from the Modern Web** - Ryan Mitchell
  - **Hands-On Web Scraping with Python: Perform advanced scraping operations using various Python libraries and tools such as Selenium, Regex, and others** - Anish Chapagain
  - **Python Automation Cookbook: 75 Python Automation Ideas for Web** - Jaime Buelta
- Frameworks and libraries:
  
  
  
  - [Scrapy](https://scrapy.org/) - a framework that allows you to organize scraping so that the code is easy to maintain and modular. It is used by many websites that collect data from other sites and many companies to build reports on the state of the market.
  - [MechanicalSoup](https://mechanicalsoup.readthedocs.io/en/stable/) - a library that aims to simulate interaction with a page without using a browser. The library allows you to create an object pretending to be a browser that can click links and send forms. It just can't handle JavaScript - for some sites Selenium is still indispensable.
